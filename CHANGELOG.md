# 更新日志

## v1.1.0 - 2026-02-02

### 🎉 新增功能

#### 配置管理系统
添加了完整的配置保存、加载和分享功能，让你的应用选择可以轻松备份和共享。

**主要特性：**

1. **💾 保存配置**
   - 保存当前应用选择到浏览器本地存储
   - 支持自定义配置名称
   - 自动保留最近 5 个配置
   - 包含发行版、应用列表和时间戳

2. **📥 导出为文件**
   - 将配置导出为 JSON 文件
   - 便于备份到云盘或版本控制系统
   - 适合团队共享标准环境配置
   - 文件格式简单易读，支持手动编辑

3. **📤 导入配置**
   - 从 JSON 文件快速加载配置
   - 自动验证文件格式和数据完整性
   - 智能过滤不支持的应用
   - 导入后自动保存到历史记录

4. **🔗 分享链接**
   - 生成包含配置的分享 URL
   - 配置信息编码在 URL 参数中
   - 无需服务器存储，完全客户端实现
   - 一键复制到剪贴板，方便分享

5. **🕐 最近配置**
   - 快速访问最近保存的配置
   - 显示配置名称、发行版、应用数量和日期
   - 一键加载历史配置

**新增文件：**
- `src/lib/configManager.ts` - 配置管理核心功能
- `src/components/config/ConfigManager.tsx` - 配置管理 UI 组件
- `CONFIG_GUIDE.md` - 详细使用指南

**改进的文件：**
- `src/hooks/useLinuxInit.ts` - 添加 URL 参数加载支持
- `src/app/page.tsx` - 集成配置管理组件
- `README.md` - 更新功能说明

### 💡 使用场景

1. **新机器配置**：在新安装的系统上快速恢复软件环境
2. **团队协作**：技术负责人创建标准配置分享给团队
3. **备份恢复**：定期导出配置防止配置丢失
4. **多环境管理**：为不同用途维护不同的软件配置

### 🎯 适合 GitHub Pages

所有功能都是纯前端实现，无需后端服务器：
- ✅ 配置存储在浏览器本地
- ✅ URL 分享完全静态
- ✅ 不需要数据库
- ✅ 完美适配 GitHub Pages 部署
